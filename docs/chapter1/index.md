# 项目概览与总体方案

> [!NOTE]
> 本章节站在「整体方案设计」的角度，回答三个问题：  
> 1）为什么要把 OpenClaw 部署到实体机器人上？  
> 2）我们采用了怎样的整体架构与路线？  
> 3）如果你是第一次接触，本项目，大致应该如何上手？

## 1. 项目动机：从桌面 Agent 到具身智能

- OpenClaw 在桌面端爆火，本质是「大模型 + 电脑文件 / 应用控制接口」，释放了强大的 Agent 能力；
- Xbotics 社区希望进一步探索：**当我们把机器人本体开放给大模型时，能否用 OpenClaw 框架实现更加丰富的物理世界交互？**
- 围绕这个问题，本项目提出了一条具身智能发展路径：
  - 大模型（LLM / VLM / VLA）  
  + OpenClaw / Picoclaw 框架  
  + 机器人本体（底盘 + 机械臂 + 机械爪 + 传感器）  
  + 一系列任务与应用场景。

目标是：**让大模型不仅能「用电脑」，还可以「开门、移动、抓东西」——真正走向 physical AI。**

## 2. 整体架构概览

从高层看，本项目的核心组件包括：

- **智能层**：大模型（如通过 OpenClaw / Picoclaw 接入的 LLM / VLA），负责理解自然语言、规划任务；
- **中间层（OpenClaw / Picoclaw）**：  
  - 把高层意图转化为「技能调用」与 API 调用；  
  - 在本项目中，最关键的技能之一就是「机器人控制技能」（如 HomeBot skill）。
- **机器人控制层（HomeBot 等）**：  
  - 管理底盘、机械臂、机械爪与传感器；  
  - 提供简洁的控制接口（如移动、转向、夹爪开合、视觉采集等）。
- **硬件层**：  
  - 机器人底盘、机械臂、夹爪、摄像头等。

后续章节会分别从「硬件与软件部署实战」「活动与课程 / 赛事」等角度展开。

## 3. 典型平台：HomeBot + OpenClaw

在当前阶段，我们将 **HomeBot** 作为主要的示例平台之一：

- 基于宜家小车 + 全向轮 Kiwi 底盘 +（可选）机械臂；
- 使用笔记本 / 树莓派等作为控制器；
- 通过 HomeBot 项目实现底盘、机械臂、视觉等统一控制；
- 再通过 OpenClaw / Picoclaw 将其纳入「大模型 Agent」的控制能力之下。

你可以在「硬件与软件部署实战」章节中看到：

- 如何搭建这套硬件平台；
- 如何安装与配置 HomeBot；
- 如何将其接入 OpenClaw / Picoclaw，实现语音 / 文本控制。

## 4. 快速上手路线（鸟瞰）

如果你第一次接触这个项目，推荐的路径是：

1. **只读理解**：  
   - 通读本章节，理解整体架构与目标；
   - 了解 HomeBot + OpenClaw 的组合关系。
2. **环境与软件准备**：  
   - 准备一台开发主机（如 Ubuntu / 带 WSL 的 Windows）；  
   - 安装 OpenClaw / Picoclaw；  
   - 克隆并安装 HomeBot（详见部署章节）。
3. **硬件搭建与连通性验证**：  
   - 按照教程完成底盘、机械臂、机械爪与控制器连接；  
   - 使用 HomeBot 提供的 Web 控制端确认运动与视觉正常。  
4. **接入 OpenClaw / Picoclaw**：  
   - 为 HomeBot 安装控制技能；  
   - 尝试通过自然语言指令让机器人移动 / 抓取 / 观察。  
5. **设计你自己的任务或课程项目**：  
   - 在此基础上扩展新的任务场景，并沉淀为新的实战案例。
