# 硬件与软件部署实战（HomeBot 等平台）

> [!NOTE]
> 本章节聚焦于一件事：**把 OpenClaw / Picoclaw 真正部署到实体机器人上**。  
> 这里以 HomeBot 平台为主线示例，你可以在此基础上拓展到其他机器人平台。

## 1. 总体部署流程概览

结合《🦞+🤖：将OpenClaw部署到机器人上！详细开发指导【xbotics】》，整体流程可以概括为：

1. 明确项目目标与场景（为什么要做这个机器人化项目）；  
2. 准备硬件（底盘、机械臂 / 机械爪、控制器、传感器等）；  
3. 安装与配置 OpenClaw / Picoclaw；  
4. 安装与配置机器人控制软件（如 HomeBot）；  
5. 打通本地控制链路（Web 控制端等）；  
6. 将机器人控制能力以「技能」的形式接入 OpenClaw / Picoclaw；  
7. 设计与验证具体任务（如移动、跟随、抓取等）。

## 4. 软件部署：OpenClaw 与 HomeBot

> 这一节对应原文中的「软件部署步骤」，可以将命令和代码片段按小节搬运进来。

### 4.1 OpenClaw 安装与部署（概览）

- 根据 OpenClaw 官方教程安装依赖与运行环境；  
- 如使用飞书等平台，可简要说明部署方式；  
- 强调后续会通过 OpenClaw / Picoclaw 调用机器人控制技能。

### 4.2 HomeBot 安装与配置

1. 克隆仓库：  
   ```bash
   git clone https://github.com/choco-robot/HomeBot.git
   ```
2. 在项目根目录进行本地安装（建议虚拟环境）：  
   ```bash
   python -m pip install -e .
   ```
3. 配置文件说明：  
   - 主要配置集中在 `software/src/config/configs.py`；  
   - 至少要正确设置底盘与机械臂的串口（`serial_port`、`baudrate` 等）；  
   - 其他参数可参考文档 `docs/配置修改说明.md`。

### 4.3 启动系统服务

- 在项目根目录执行：  
  ```bash
  python software/start_system.py
  ```  
- 会自动启动：
  - 底盘与机械臂控制服务；  
  - 视觉服务（摄像头图像流）；  
  - Web 控制端服务（默认端口 5000，可在配置中修改）。

## 5. 控制与交互方式

### 5.1 Web 控制端（局域网）

- 在与机器人同一局域网的电脑 / 手机浏览器中，访问：  
  `http://<机器人 IP>:5000`；  
- 控制说明：
  - 左手摇杆：控制底盘前进 / 后退 / 转向；  
  - 右手摇杆：控制机械臂底座旋转、抬升等；  
  - 按钮：紧急停止、归位、夹爪开合、开启 / 关闭人体跟随模式等；  
- 安全注意事项：
  - 紧急停止会立刻刹车并锁定控制，需通过「归位」解锁；  
  - 在人体跟随模式下，手动控制会有限制，避免冲突。

### 5.2 AI 助手控制（OpenClaw / Picoclaw）

- 确保 OpenClaw / Picoclaw 与 HomeBot 处于同一局域网；  
- 通过一次自然语言指令，为 OpenClaw 安装 HomeBot 控制技能，例如：  
  ```text
  帮我安装这个技能 https://github.com/choco-robot/homebot/tree/main/skills/homebot
  ```  
- 之后可以用自然语言控制机器人，例如：  
  - 「让机器人前进 20 厘米」  
  - 「右转 90 度」  
  - 「机器人看到了什么？」  
- 由 Picoclaw / OpenClaw 自动解析语义并调用 HomeBot 的底盘、机械臂、视觉等接口完成操作。
